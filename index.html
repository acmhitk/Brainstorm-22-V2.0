Sova and Chocolates

Problem Link : Link
Author : Ankur Kayal
Editorialist : Shahil Singh

Difficulty

Cakewalk

Prerequisites:

Implementation

Problem

Sova has NNN chocolates available with him. Each of the NNN chocolates has an irritation value AiA_iAi​. Sova likes eating chocolates but since the chocolates have an irritation value he avoids eating them. Fortunately, he can throw away any 111 chocolate but has to eat the rest N−1N - 1N−1 chocolates. Can you help Sova calculate the minimum amount of irritation value that he has to suffer if he strategically throws away 111 chocolate?

Explanation

Since Sova can throw any 111 chocolate and he has to make sure he suffers minimum amount of irritation value, it is always beneficial for him to throw away the chocolate with maximum irritation value. Therefore we can find the sum of all irritation values of the chocolates and subtract the greatest irritation value of all chocolates.

Implementation notes: The irritation values of all the chocolates are not unique, i.e. 222 or more chocolates can have same irritation value. And if the number of chocolates equals 111 then he doesn’t have to eat any chocolate.

Time Complexity

O(N)O(N)O(N)

Space Complexity

O(1)O(1)O(1)

Setters Solution (C++)

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(0); cin.tie(0);

    int N;
    cin >> N;

    vector<int> A(N);
    for(int i = 0; i < N; i++) {
        cin >> A[i];
    }

    int sum = 0;
    for(int i = 0; i < N; i++) {
        sum += A[i];
    }

    int maxA = A[0];
    for(int i = 0; i < N; i++) {
        maxA = max(maxA, A[i]);
    }

    sum -= maxA;

    cout << sum << '\n';
}

Jett in the Infinite Grid

Problem Link : Link
Author : Ankur Kayal
Editorialist : Ankur Kayal

Difficulty

Easy

Prerequisites:

Implementation, GCD

Problem

Brimstone trapped Jett in an Infinite 2D World as punishment for using super dash in the VALORANT Protocol. The Infinite 2D World is supposed to be an inescapable Infinite 2D Coordinate Grid where everything lies on an integral coordinate point!

However, being Agent 10, Jett figured out that there is an exit portal at coordinate (P, Q). Expecting this, Brimstone put another restriction. At any given time, if Jett is at coordinate (a, b) she can only dash to coordinates - (a + b, b) or (a - b, b) or (a, b + a) or (a, b - a).

Jett finds herself on the coordinates (X, Y) initially. Given these conditions can you help Jett answer if she can reach the exit portal or if she is to roam this 2D World for eternity?

Explanation

Let us look carefully the coordinates that Jett can dash from coordinate points (a,b)(a, b)(a,b). The coordinate points are (a+b,b)(a + b, b)(a+b,b), (a−b,b)(a - b, b)(a−b,b), (a,b+a)(a, b + a)(a,b+a), (a,b−a)(a, b - a)(a,b−a). If Jett continues to perform dashes repeatedly, then Jett will reach a special coordinate (0,g)(0, g)(0,g) or (g,0)(g, 0)(g,0), where ggg is the greatest common divisor of aaa and bbb. A more detailed explanation here: link.

Now in the problem we have the coordinate points (X,Y)(X, Y)(X,Y) and (P,Q)(P, Q)(P,Q). Now lets convert both the coordinate points to the format discussed. So (X,Y)(X, Y)(X,Y) after a series of operations becomes (g,0)(g, 0)(g,0) and (P,Q)(P, Q)(P,Q) after a series of operations becomes (g′,0)(g', 0)(g′,0), where g=gcd(X,Y)g = gcd(X, Y)g=gcd(X,Y) and g′=gcd(P,Q)g' = gcd(P, Q)g′=gcd(P,Q). Now Jett can escape if and only if g=g′g = g'g=g′. This is so because if g=g′g = g'g=g′ then (g,0)=(g′,0)(g, 0) = (g', 0)(g,0)=(g′,0). Then from this coordinates we can trace back to the original coordinates (X,Y)(X, Y)(X,Y) and (P,Q)(P, Q)(P,Q).

Therefor Jett can escape if and only if gcd(P,Q)=gcd(X,Y)gcd(P, Q) = gcd(X, Y)gcd(P,Q)=gcd(X,Y).

Time Complexity

O(log(max(X,Y,A,B))O(log(max(X,Y,A,B))O(log(max(X,Y,A,B)) per test case

Space Complexity

O(1)O(1)O(1)

Setters Solution (C++)

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(0); cin.tie(0);

    int T;
    cin >> T;

    while(T--) {
        int X, Y, P, Q;
        cin >> X >> Y >> P >> Q;

        if(__gcd(X, Y) == __gcd(P, Q)) {
            cout << "Escape" << '\n';
        } else {
            cout << "Eternal Punishment" << '\n';
        }
    }
}

Setters Solution (Java)

import java.util.Scanner;

class Solution {
    static int gcd(int a, int b) {
        if(a == 0 && b == 0) {
            return 1;
        }
        if(b == 0) {
            return a;
        }
        return gcd(b, a % b);
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int T = sc.nextInt();

        for(int tt = 0; tt < T; tt++) {
            int X = sc.nextInt();
            int Y = sc.nextInt();
            int P = sc.nextInt();
            int Q = sc.nextInt();

            if(gcd(X, Y) == gcd(P, Q)) {
                System.out.println("Escape");
            } else {
                System.out.println("Eternal Punishment");
            }
        }
    }
}

Search for Agent 18

Problem Link : Link
Author : Ankur Kayal
Editorialist : Ankur Kayal

Difficulty

Medium

Prerequisites:

Bitwise Operations, Prefix sums

Problem

In the VALORANT Protocol, Sage is in search of Agent 18. She has shortlisted Chamber as Agent 18. But before being recruited Chamber needs to solve a problem set by Brimstone, the Leader of Valorant Protocol.

Chamber was given an array AAA of size NNN, each of the elements denoted by AiA_iAi​. We define bitwise AND of two elements of aaa and bbb as a & ba \: \& \: ba&b. He needs to find the longest subarray such that the bitwise AND of all the elements of the subarray is greater than 0.

Chamber tried day and night but could not solve it. He is very eager to join the VALORANT Protocol but if he cannot solve then he would not be allowed to join as Agent 18. Can you help Chamber solve the problem?

Note: The subarray of an element is obtained by deleting some elements from the beginning and some elements from the end.

Explanation

Let us explore what bitwise AND (&\&&) of two numbers aaa and bbb mean. a & ba \: \& \: ba&b is non-zero only when aaa and bbb have atleast 111 bit common in the binary representation of aaa and bbb. Since a≤231a \leq 2^{31}a≤231 in the problem, there are 313131 bits in the numbers. And we need to calculate the longest subarray with bitwise AND of the numbers greater than 000.

So we can follow the following strategy: Iterate over the 313131 bits ranging from 000 to 303030 both inclusive and then we can form a modified array AAA, marking AiA_iAi​ as 111 if the current bit is set otherwise mark it as 000. Now the question transforms to a more simplified question: Calculate the longest subarray with only 111. This is a known problem which can be solved by maintaining the current length of the subarray when we get 111 and resetting it when we get 000. Detailed explanation can be found here: link.

Now we need to perform this task 313131 times and maximise our answer to get the longest subarray.

Time Complexity

O(31 ∗ N)O(31 \: * \: N)O(31∗N)

Space Complexity

O(N)O(N)O(N)

Setters Solution (C++)

#include <bits/stdc++.h>
using namespace std;

int main() {
    ios_base::sync_with_stdio(0); cin.tie(0);

    int T;
    cin >> T;

    while(T--) {
        int N;
        cin >> N;

        vector<int> A(N);

        for(int i = 0; i < N; i++) {
            cin >> A[i];
        }

        int largestSubarray = 0;

        for(int bit = 0; bit < 31; bit++) {
            int currentSubarrayLength = 0;

            for(int i = 0; i < N; i++) {
                if(A[i] >> bit & 1) {
                    currentSubarrayLength++;
                } else {
                    largestSubarray = max(largestSubarray, currentSubarrayLength);
                    currentSubarrayLength = 0;
                }
            }

            largestSubarray = max(largestSubarray, currentSubarrayLength);
        }

        cout << largestSubarray << '\n';
    }
}

Search for Agent 19

Problem Link : Link
Author : Ankur Kayal
Editorialist : Ankur Kayal

Difficulty

Hard

Prerequisites:

Trees, DFS, DFS Tree, Euler Tour, Binary Lifting, LCA, Mo’s ALgorithm

Problem

Chamber was recruited by Sage as Agent 18. Now Sage is searching for suitable candidates for Agent 19. Neon was selected as a suitable candidate. As usual, she must pass Brimstone’s test to be a part of the VALORANT protocol.

Neon was given a tree with NNN nodes (numbered 111 through NNN). For each valid node iii, the ithi^{th}ith node have VALORANT point ViV_iVi​ associated with it. The nodes are connected by N−1N - 1N−1 edges such that there exists a unique path from any one node to all other nodes. Let us consider two nodes xxx and yyy. Neon is also given a certain value of RADIANITE point rrr for those pairs of vertices xxx and yyy. Consider all the nodes on the path from xxx to yyy. Let the number of nodes on that path be ppp. Now she has to find is there a way to choose p−1p - 1p−1 vertices such that the XOR of all the VALORANT points of those vertices equals rrr. If there exists a way she has to say “YES” else she has to say “NO” without quotes.

However Brimstone is not so lenient. He reframed the question and asked Neon to answer the question for QQQ pairs of vertices xxx and yyy. Can you help Neon solve the problem so that she is recruited by Brimstone as soon as possible and can join VALORANT protocol as Agent 19?

Explanation

We have to find if we can achieve the following radianite point rrr if we calculate the XOR all except 111 node on the path from nodes xxx to yyy. Let the discarded node be did_idi​.

Now,
XOR (all nodes except 111 on the path from nodes xxx to yyy) = rrr
XOR (all nodes on the path from nodes xxx to yyy) = rrr xor VdiV_{d_i}Vdi​​.
(((XOR (all nodes on the path from nodes xxx to yyy)))) xor rrr === rrr xor VdiV_{d_i}Vdi​​ xor rrr
(((XOR (all nodes on the path from nodes xxx to yyy)))) xor rrr === VdiV_{d_i}Vdi​​

Therefore the problem gets converted to a different problem, we have to find whether a node iii with Valorant point ViV_iVi​ exists on the path from xxx to yyy where the following condition is satisfied: (((XOR (all nodes on the path from nodes xxx to yyy)))) xor rrr === ViV_iVi​.

To find whether a node exists with a certain Valorant point on the path from xxx to yyy, we first flatten the tree with the help of DFS and Euler Tour. Then we use Mo’s algorithm on trees to solve whether a node with certain Valorant point exists on the path or not. A detailed explanation about this technique can be found here: link.

Finally if we find such a node we print “YES” else “NO”.

Time Complexity

O(N∗logN+Q∗logN+Q∗logQ+Q∗sqrt(N))O(N * logN + Q * logN + Q * logQ + Q * sqrt(N))O(N∗logN+Q∗logN+Q∗logQ+Q∗sqrt(N))

Notes on time complexity :

  1. Preprocessing to find LCA between two nodes xxx and yyy takes O(N∗logN)O(N * logN)O(N∗logN).
  2. Preparing the queries to solve it offline takes O(Q∗logN)O(Q * logN)O(Q∗logN).
  3. Sorting all the queries takes O(Q∗logQ)O(Q * logQ)O(Q∗logQ).
  4. Finally, answering all the queries offline using Mo’s algorithm takes O(Q∗sqrt(N))O(Q * sqrt(N))O(Q∗sqrt(N)).

Space Complexity

O(NlogN)O(NlogN)O(NlogN)
